[ ] ETAPA 14 — Stable Diffusion Worker Base

Objetivo:
Criar um serviço local próprio da KIT para geração e edição básica de imagens via Stable Diffusion, sem uso de ComfyUI, Forge ou WebUI.

Inclui:
- Criar worker Python local usando Diffusers
- Criar API HTTP local para o worker
- Criar endpoints mínimos:
  - /health
  - /models
  - /txt2img
  - /img2img
  - /inpaint
- Backend da KIT deve consumir essa API local
- Canvas deve conseguir solicitar geração usando o artboard ativo como contexto
- Salvar outputs em pasta dedicada, como /output/sd
- Registrar no .kia:
  - prompt positivo
  - prompt negativo
  - checkpoint usado
  - LoRA usado
  - seed
  - modo usado
  - width
  - height
  - sampler/scheduler
  - arquivo de saída gerado

Arquitetura suportada:
- Checkpoints SD 1.5
- Checkpoints SDXL
- Modelos locais .safetensors e .ckpt
- Detecção automática do tipo do modelo quando possível
- Permitir seleção manual do tipo do modelo se a detecção falhar

Regras de carregamento:
- Carregar modelo sob demanda (lazy load)
- Não manter dois checkpoints pesados carregados ao mesmo tempo
- Antes de carregar novo checkpoint:
  - descarregar o anterior
  - limpar cache CUDA
- Não usar refiner nesta fase
- Não implementar ControlNet nesta fase
- Não implementar upscale/highres fix nesta fase

Execução em GPU:
- Priorizar sempre CUDA na RTX disponível
- Usar fp16 quando suportado
- Usar xformers se disponível
- Aplicar otimizações de memória compatíveis com Diffusers
- Limpar VRAM após geração quando necessário

Regra de proporção com o Canvas:
- O worker não deve gerar usando resolução fixa sem contexto
- Antes de gerar, o backend deve consultar o Canvas ativo e obter:
  - preset atual do artboard
  - largura do artboard
  - altura do artboard
  - proporção atual
- A geração deve herdar a proporção do artboard ativo
- O tamanho final exportado continua sendo responsabilidade do Canvas
- O tamanho de geração é responsabilidade do SD Worker

Presets por proporção:
- instagram_post → 1:1
- instagram_story → 9:16
- instagram_reels → 9:16
- facebook_post → 1:1 ou 4:5
- youtube_thumb → 16:9
- custom → proporção derivada do artboard ativo

Resoluções padrão por arquitetura:
SD 1.5:
- 1:1 → 512x512
- 4:5 → 512x640
- 9:16 → 512x912
- 16:9 → 768x432

SDXL:
- 1:1 → 1024x1024
- 4:5 → 1024x1280
- 9:16 → 1024x1792
- 16:9 → 1344x768

Fallbacks:
- SDXL deve ter fallback para resolução menor se faltar VRAM
- Width e Height podem ser alterados manualmente, mas por padrão devem seguir o artboard ativo
- Após gerar, a imagem deve ser inserida no Canvas e adaptada ao artboard
- Inicialmente usar modo de encaixe padrão "cover"

Controles obrigatórios do painel Stable Diffusion:
Campos principais:
- Prompt positivo
- Prompt negativo

Seleção de modelo:
- Checkpoint
  - Lista expansível lateral
  - Exibir nome do modelo
  - Se possível, exibir miniatura/preview
  - Ao selecionar, fechar menu automaticamente

- LoRA
  - Lista expansível lateral
  - Exibir nome do LoRA
  - Se possível, exibir miniatura/preview
  - Permitir selecionar e remover LoRA
  - Ao selecionar, fechar menu automaticamente

Configurações de geração:
- Sampling method
- Schedule type
- Sampling steps
- Width
- Height
- CFG Scale
- Seed
  - Valor padrão: -1
  - -1 significa seed aleatória

Modos:
- txt2img
- img2img
- Sketch
- Inpaint
- Inpaint sketch
- Inpaint upload
- Batch

Controles específicos de img2img/inpaint:
- CFG Scale
- Denoising strength

Batch:
- Batch count fixo em 1 na Fase 1
- Batch size fixo em 1 na Fase 1

Regra:
- Na Fase 1, gerar apenas uma imagem por vez
- O objetivo é reduzir consumo de VRAM e evitar travamentos

Modo avançado:
- Exibir os nomes técnicos reais dos schedulers do Diffusers
- Exemplos:
  - DPMSolverMultistepScheduler
  - EulerDiscreteScheduler
  - EulerAncestralDiscreteScheduler
  - DDIMScheduler
  - LMSDiscreteScheduler

Critério de conclusão:
A KIT consegue selecionar um checkpoint local, usar a proporção do artboard ativo, gerar imagem via worker próprio e inserir o resultado no Canvas.

Não incluir ainda:
- vídeo com IA
- geração de áudio por diffusion
- ControlNet
- upscaling avançado
- highres fix
- refiner
- batch maior que 1
